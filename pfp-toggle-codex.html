PFP Dark Mode
Toggle System

v1.0 Vanilla JS 4 Assets No Dependencies
01 — Overview

What this system does

The PFP toggle renders a static image by default. When the user switches theme, a GIF plays exactly once through the transition animation, then is replaced by a new static frame matching the target theme. No looping. No flicker. Clean state.

Page load
static_light.png — default, no glasses
→ Dark mode
gif_to_dark.gif — plays once (4.9s), loop=1
Dark idle
static_dark.png — glasses on, static
→ Light mode
gif_to_light.gif — plays once (4.5s), loop=1
Light idle
static_light.png — back to start
02 — File Structure

Assets to deploy

Keep all 4 assets in the same directory as the page, or update the paths in the JS config object.

your-site/ ├─ gif_to_dark.gif # 6.4 MB — light → dark transition ├─ gif_to_light.gif # 5.8 MB — dark → light transition ├─ static_light.png # 203 KB — default idle frame ├─ static_dark.png # 179 KB — dark idle frame └─ index.html # your page
03 — Markup

HTML structure

A single <img> handles all states. The frame container should have overflow: hidden and a fixed size so the swap is invisible.

HTML 
<!-- PFP container — shape it however you want -->
<div class="pfp-frame">
  <img
    id="pfp"
    src="static_light.png"
    alt="Profile picture"
    draggable="false"
  />
</div>

<!-- Your existing dark mode toggle, any element -->
<button id="theme-toggle">Toggle theme</button>
Note — The id="pfp" is what the JS hooks into. Rename it in both places if needed.

05 — JavaScript

The toggle logic

Drop this before the closing </body> tag. The only values you'll ever need to touch are in the assets and GIF_DURATION objects at the top.

JavaScript 
<script>
  // ── CONFIG ───────────────────────────────────────
  const assets = {
    staticLight : 'static_light.png',
    staticDark  : 'static_dark.png',
    gifToDark   : 'gif_to_dark.gif',
    gifToLight  : 'gif_to_light.gif',
  };

  // Duration of each GIF in milliseconds
  // If the static frame snaps too early/late, adjust here
  const GIF_DURATION = {
    toDark  : 4900,   // gif_to_dark.gif  (~4.9s)
    toLight : 4500,   // gif_to_light.gif (~4.5s)
  };
  // ─────────────────────────────────────────────────

  const pfp    = document.getElementById('pfp');
  const toggle = document.getElementById('theme-toggle');
  const html   = document.documentElement;

  let isDark    = false;
  let animating = false;

  // Cache-bust forces the GIF to replay from frame 1 every time
  function playGif(src) {
    return `${src}?t=${Date.now()}`;
  }

  toggle.addEventListener('click', () => {
    if (animating) return;   // block double-clicks during animation
    animating = true;
    toggle.disabled = true;

    if (!isDark) {
      // ── Light → Dark ──
      pfp.src = playGif(assets.gifToDark);

      setTimeout(() => {
        pfp.src         = assets.staticDark;
        isDark          = true;
        animating       = false;
        toggle.disabled = false;
        html.setAttribute('data-theme', 'dark');
      }, GIF_DURATION.toDark);

    } else {
      // ── Dark → Light ──
      pfp.src = playGif(assets.gifToLight);

      setTimeout(() => {
        pfp.src         = assets.staticLight;
        isDark          = false;
        animating       = false;
        toggle.disabled = false;
        html.setAttribute('data-theme', 'light');
      }, GIF_DURATION.toLight);
    }
  });
</script>
06 — Timing Calibration

Tuning the freeze point

The GIF plays for exactly GIF_DURATION ms, then the static PNG replaces it. If the freeze feels early or late, adjust by ±200ms at a time.

Symptom Cause Fix
Static frame snaps in too early Duration too short Increase GIF_DURATION.toDark by 200ms
GIF loops before snapping to static Duration too long Decrease the value by 200ms
Visible flicker on swap Static PNG not preloaded See preload section below
GIF replays instead of playing fresh Browser cached old src playGif() handles this — ensure it's called
07 — Preloading

Eliminate flicker on first swap

GIFs are large. Without preloading, the first toggle shows a blank frame while the browser fetches the file. Add these tags inside <head> to silently preload both GIFs on page load.

HTML — inside <head> 
<!-- Preload transition GIFs so first toggle is instant -->
<link rel="preload" as="image" href="gif_to_dark.gif"  />
<link rel="preload" as="image" href="gif_to_light.gif" />
Warning — Preloading 12MB of GIFs on page load will delay LCP on slow connections. If your site is performance-sensitive, consider lazy preloading after the first user interaction instead.
JS — lazy preload after first interaction 
// Preload both GIFs only after the user first hovers the toggle
let preloaded = false;

toggle.addEventListener('mouseenter', () => {
  if (preloaded) return;
  [assets.gifToDark, assets.gifToLight].forEach(src => {
    const img = new Image();
    img.src = src;
  });
  preloaded = true;
}, { once: true });    // fires only once
08 — Integration

Hooking into an existing toggle

If your site already has a dark mode system (Tailwind, next-themes, a custom class toggle), you don't need a new button. Just call the PFP logic from wherever your theme changes.

Tailwind / class-based

JS 
// Replace your existing toggle handler with this wrapper
function setTheme(dark) {
  // Your existing logic
  document.documentElement.classList.toggle('dark', dark);
  localStorage.setItem('theme', dark ? 'dark' : 'light');

  // PFP logic — append to your existing handler
  if (animating) return;
  animating = true;

  const gif = dark ? assets.gifToDark : assets.gifToLight;
  const end = dark ? assets.staticDark : assets.staticLight;
  const dur = dark ? GIF_DURATION.toDark : GIF_DURATION.toLight;

  pfp.src = playGif(gif);
  setTimeout(() => {
    pfp.src   = end;
    animating = false;
  }, dur);
}

next-themes (React / Next.js)

JSX 
import { useTheme } from 'next-themes'
import { useRef, useState } from 'react'

const GIF_DURATION = { toDark: 4900, toLight: 4500 }

export function ProfilePic() {
  const { theme, setTheme } = useTheme()
  const [src, setSrc] = useState('/static_light.png')
  const animating = useRef(false)

  const handleToggle = () => {
    if (animating.current) return
    animating.current = true
    const dark = theme !== 'dark'
    setTheme(dark ? 'dark' : 'light')
    setSrc(`${dark ? '/gif_to_dark.gif' : '/gif_to_light.gif'}?t=${Date.now()}`)
    setTimeout(() => {
      setSrc(dark ? '/static_dark.png' : '/static_light.png')
      animating.current = false
    }, dark ? GIF_DURATION.toDark : GIF_DURATION.toLight)
  }

  return (
    <div className="pfp-frame">
      <img src={src} alt="pfp" onClick={handleToggle} />
    </div>
  )
}
09 — Deploy Checklist

Before going live

  • All 4 asset files are in the correct directory
  • Asset paths in the assets config object match actual filenames
  • Tested both transitions — light→dark and dark→light
  • Timing feels natural — no early snap, no loop bleed
  • Button is disabled during animation (double-click protection is in)
  • PFP is not cropped incorrectly inside the frame — tuned object-position
  • Preloading strategy chosen (eager in <head> or lazy on hover)
  • Tested on mobile — tap, not hover
